# 文件树修改状态快速定位条 - 设计规范

## 功能概述

在文件浏览区（FileTree）右侧添加一个独立的侧边状态条，用小色块标记所有有Git修改的文件在整个文件树中的位置，支持点击快速跳转定位。

## 背景和动机

在大型仓库中，文件树内容很长，当有多个修改文件分散在不同目录时，用户需要滚动很久才能找到所有修改文件。这个状态条可以让用户：

1. 一眼看出修改文件在整个目录树中的分布
2. 点击即可快速跳转到对应位置
3. 提高代码评审和提交操作的效率

## 设计方案

### 布局结构

```
┌─────────────────────────────────┐ ┌──────┐
│ 标签栏：文件 暂存区 收藏 🔍       │ │      │
├─────────────────────────────────┼─┤      │
│                                 │ │  状  │
│  文件树列表（可滚动）            │ │  态  │
│                                 │ │  条  │
│                                 │ │      │
│                                 │ │      │
└─────────────────────────────────┴─┴──────┘
```

- **位置**：文件树列表区域的最右侧，紧贴内容
- **宽度**：固定 10px
- **高度**：与文件树列表可视区域高度一致
- **背景色**：浅灰色（适配深色模式）

### 标记规则

- 每个**非目录**修改文件对应一个 `8px × 8px` 的小色块
- 色块颜色与文件树中文件名的状态颜色保持一致：
  - 新增文件：绿色 `#009900`
  - 删除文件：红色 `#cc0000`
  - 修改文件：蓝色 `#0066cc`
  - 冲突文件：橙色 `#ff8800`
- 色块使用绝对定位，`top` 值根据文件在整个文件树中的相对位置计算得出：

```
top = (fileIndex / totalFiles) * containerHeight
```

其中：
- `fileIndex`：文件在所有渲染文件中的索引位置
- `totalFiles`：所有渲染文件的总数
- `containerHeight`：状态条容器高度（等于文件树可视区域高度）

### 交互设计

1. **点击跳转**：点击任意色块 → 文件树滚动到对应文件位置并选中该文件
2. **悬浮提示**：鼠标悬停在色块上 → 显示 tooltip，内容为 `文件路径 (状态)`
3. **自动更新**：当 Git 状态变化、搜索过滤、标签切换时，状态条自动重新计算并更新标记

### 组件设计

新建组件 `FileTreeStatusBar.tsx`：

```
src/renderer/components/FileTreeStatusBar.tsx
```

**Props 接口：**

```typescript
interface FileTreeStatusBarProps {
  gitStatus: GitStatus;
  visibleFiles: FileNode[];  // 当前可见的所有文件节点（已过滤）
  onJumpToFile: (file: FileNode) => void;  // 跳转回调
}
```

**修改现有文件：**

- `FileTree.tsx`：引入状态条组件，调整布局，传递必要 props
- `App.tsx`：无变更（状态条在 FileTree 组件内部）

### 响应式和性能

- 状态条宽度固定，不影响文件树拖动调整宽度功能
- 仅在 `gitStatus` 变化或可见文件变化时重新计算标记，避免不必要渲染
- 当没有修改文件时，状态条依然显示（空白）保持布局稳定

### 主题适配

- 亮色模式：状态条背景 `#eaeaea`
- 深色模式：状态条背景 `#4a4a4a`
- 色块颜色与亮色/深色模式兼容，保证对比度足够
